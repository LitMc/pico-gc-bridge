# GCコントローラ解析 - 目的と背景

## 目的
本プロジェクトは、GameCubeコントローラのJoybus通信をRP2040(Pico)でリアルタイムに中継しつつ、
入力値（特にアナログスティック）を必要に応じて変換・補正することで、接続先環境で生じる入力問題を緩和することを目的とする。

## 背景（なぜ必要か）
- Joybusはタイミング・フォーマットが厳密で、ソフトウェア側の“それっぽい実装”ではズレが出やすい。
- 後方互換/移植/エミュレーション等の環境では、スティックの補正・デッドゾーン・座標変換が本来と異なる場合がある。
- そこで「物理コントローラ ↔ 本体」の間にブリッジを挟み、観測・記録・補正を可能にする。

## 成果物のイメージ
- ブリッジ動作：本体のポーリングに対し、コントローラの応答を中継（必要なら書き換え）
- 解析/計測：フレーム内容や計測データを取り、後でLUT等の補正モデルを作る
- 実運用：補正モデルを適用し、違和感を減らして操作できる状態を目指す
